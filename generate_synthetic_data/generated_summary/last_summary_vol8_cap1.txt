**Riassunto Denso – Decisioni Architetturali, Variabili e Dipendenze Tecniche (AETERNA Data Lake & Data Warehouse)**

**Decisioni Architetturali:**
- **Architettura Dati Stratificata:** Separazione netta tra Data Lake (repository scalabile per dati grezzi/semi-strutturati) e Data Warehouse (ambiente ottimizzato per analisi, reporting, supporto decisionale).
- **Edge/Fog Computing:** Raccolta dati tramite H-Node (ARM Cortex-M7) con stack MQTT 5.0 e OPC-UA, pre-elaborazione locale, buffering, firma digitale (ECDSA secp256k1).
- **Sicurezza End-to-End:** Trasporto dati cifrato (TLS 1.3), autenticazione a doppio fattore (X.509 + TPM), crittografia at-rest (AES-256 GCM), IAM granulari, audit trail immutabile.
- **Pipeline ETL Orchestrate:** Apache NiFi per Data Lake (validazione, deduplicazione, arricchimento semantico, tagging compliance Kyoto 2.0); Airflow/Kafka Connect per Data Warehouse (batch/streaming, aggregazione, KPI, normalizzazione).
- **Modellazione Dati:** Star schema (dimensioni: tempo, luogo, attore, asset, tipo transazione), tabelle fatti/dimensione, indici temporali/spaziali (PostGIS), materialized view.
- **Analisi e ML:** Batch (ARIMA/LSTM, K-Means), streaming (Isolation Forest, AutoEncoder), orchestrazione ML via Kubeflow, feature store centralizzato, enforcement policy via smart contract.
- **Visualizzazione e Reporting:** Grafana (TimescaleDB/Kafka), Power BI (compliance Kyoto 2.0), esportazione RDF/CSV/JSON.

**Variabili Introdotte:**
- **Tipologie Dati:** Sensori (JSON, Avro, Parquet), log blockchain (BET, stablecoin, fiat), SCADA, dati di mercato, metadati RDF/XML.
- **Policy di Retention:** 12 mesi dati grezzi, 36 mesi aggregati, versioning per rollback/audit.
- **QoS e Buffering:** MQTT QoS 2, OPC-UA stateful, failover locale.
- **Compliance:** Kyoto 2.0, auditabilità, data masking, RBAC.

**Dipendenze Tecniche:**
- **Tecnologie Edge:** ARM Cortex-M7, MQTT 5.0, OPC-UA, ECDSA, TPM.
- **Data Lake:** Hadoop HDFS (on-prem), Amazon S3 (cloud), Apache NiFi.
- **Data Warehouse:** TimescaleDB, Amazon Redshift, Snowflake, Airflow, Kafka Connect, PostGIS.
- **Analisi/ML:** Kubeflow, Grafana, Power BI.
- **Sicurezza:** TLS 1.3, AES-256 GCM, IAM, X.509, TPM.
- **Interoperabilità:** API GraphQL, esportazione RDF/CSV/JSON.

**Sintesi:**  
L’architettura AETERNA adotta una netta separazione tra Data Lake e Data Warehouse, con pipeline ETL avanzate, sicurezza multilivello, compliance normativa e abilitazione nativa per AI/ML e reporting regolatorio, garantendo scalabilità, resilienza, auditabilità e interoperabilità per micro-reti energetiche decentralizzate.
